# Appendix A — Data Description Maps

This appendix documents the column-level mapping between the source (OLTP) and the data mart (gold) for the two reference dimensions, **DimCustomer** and **DimProduct**. The same template can be replicated for the remaining dimensions (DimEmployee, DimLocation, DimDate, DimDeliveryMethod, DimPaymentMethod) and for the fact tables.

---

## Table 3 — Data description map of DimCustomer

| Name        | Type of table | Nr. records | Description                                                         |
|-------------|---------------|-------------|---------------------------------------------------------------------|
| DimCustomer | Dimension     | 663 (current rows) | Customers of WWI, with category (resolved from `customercategories`). SCD Type 2: historical changes are preserved via `version` / `date_from` / `date_to`. |

| Target (Data mart)   |                                |              |       | Source (OLTP)         |                          |              | ETL rules                                                                                  | Example of values                       |
|----------------------|--------------------------------|--------------|-------|-----------------------|--------------------------|--------------|-------------------------------------------------------------------------------------------|-----------------------------------------|
| **Column**           | **Description**                | **Data type**| **SCD**| **Table**            | **Column**               | **Data type**|                                                                                            |                                         |
| `CustomerKey`        | Surrogate key                  | INT (SERIAL) | 1     | —                     | —                        | —            | Auto-generated by Postgres `SERIAL` on insert. Excluded from the `to_sql` payload.        | 1, 2, 3, …                              |
| `CustomerID`         | Business key (natural)         | INT          | 1     | `customers`           | `customerid`             | int4         | Direct copy.                                                                              | 832                                     |
| `CustomerName`       | Customer name                  | VARCHAR(255) | 2     | `customers`           | `customername`           | varchar      | `clean_str` (trim, empty → NULL).                                                          | "Alvin Bollinger"                       |
| `Category`           | Customer category              | VARCHAR(255) | 2     | `customercategories`  | `customercategoryname`   | varchar      | Left-join `customers.customercategoryid = customercategories.customercategoryid`, then `clean_str`. | "Novelty Shop"                          |
| `version`            | SCD2 version counter           | INT          | —     | —                     | —                        | —            | Set to `1` on insert; incremented when an attribute change is detected (hash diff).       | 1, 2                                    |
| `date_from`          | Validity start                 | TIMESTAMP    | —     | —                     | —                        | —            | `run_at` of the load that introduced the row.                                              | 2026-05-04 22:00:00                     |
| `date_to`            | Validity end (NULL = current)  | TIMESTAMP    | —     | —                     | —                        | —            | `NULL` for current row; set on UPDATE detection to close the previous version.           | NULL or 2026-05-12 18:30:00             |

**SCD Type 2 detection.** Each silver row is hashed over `[CustomerName, Category]` (MD5 over `|`-joined values). The current gold row for the same `CustomerID` is hashed over the same columns. If hashes differ, the gold row is closed (`date_to = run_at`), and a new row is inserted with `version + 1`.

## Data lineage cheat sheet

```
public.customers ─┬─▶ bronze.customers ──▶ silver.stg_customers ──▶ gold.dimcustomer (SCD2)
                  └─▶ join on customercategoryid
public.customercategories ─▶ bronze.customercategories ─▶ (silver join only) ─▶ gold.dimcustomer

public.stockitems ─▶ bronze.stockitems ─▶ silver.stg_products ─▶ gold.dimproduct (SCD2)
```
